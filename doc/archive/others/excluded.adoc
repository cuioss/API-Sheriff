= Excluded API Gateways
:sectnums:

The following API gateways were considered but excluded from detailed evaluation.

[cols="1,2,1"]
|===
| Project | Reason for Exclusion | Last Activity

| Apiman (apiman.io)
| Effectively dormant. Last stable release 3.1.3.Final in November 2023. Only 1 commit in 2024. Apiman 4 announced as WIP but no public timeline or code visible.
| November 2023

| WSO2 API Manager
| Primarily a commercial product. Open-source edition requires heavy infrastructure (multiple databases, message brokers). Overly complex for lightweight gateway use cases.
| Active but commercial-focused

| Amazon API Gateway
| Proprietary AWS service, not open-source.
| N/A

| Google Apigee
| Proprietary Google Cloud service, not open-source.
| N/A

| MuleSoft Anypoint
| Proprietary Salesforce product, not open-source.
| N/A

| Zuul (Netflix)
| Tightly coupled to the Netflix/Spring Cloud ecosystem. Not a standalone API gateway product. Limited community outside Spring.
| Maintenance mode

| Spring Cloud Gateway
| Framework, not a product. Requires custom development for every gateway feature. No built-in management, developer portal, or plugin ecosystem.
| Active but framework only

| Envoy / Istio
| Service mesh proxies, not API gateways. Different architectural layer (L4/L7 sidecar proxy vs. edge API gateway). Envoy can be used as a building block but requires significant custom integration.
| Active but different scope

| Express Gateway
| Dormant. Last release in 2019. Repository archived.
| 2019 (archived)

| Fusio
| API management platform focused on API development, not gateway proxying. Different scope.
| Active but different scope

| Caddy
| General-purpose web server, not a dedicated API gateway. JWT/auth and rate limiting require custom-built binary via xcaddy plugin system. No request aggregation or transformation.
| Active but different scope

| Envoy Gateway
| Standalone mode is new and less documented. Configuration uses Kubernetes Gateway API resource model even outside K8s (verbose). Heavier than pure Go alternatives (C++ Envoy process). Global rate limiting requires Redis.
| Active but Kubernetes-focused

| Lura (Linux Foundation)
| Framework/library, not a ready-to-run product. Requires writing Go code. The usable product built on Lura is KrakenD CE (already evaluated separately).
| Active but framework only

| Pingora (Cloudflare)
| Rust framework/library, not a gateway product. Requires writing Rust code for routing, auth, and all middleware. No declarative config. Designed for teams building custom proxy infrastructure.
| Active but framework only

| Pingap
| Promising Rust-based gateway built on Pingora with built-in plugins. However, primarily a single-developer project with high bus-factor risk and limited documentation.
| Active but immature
|===
