== Architecture

* *Proxy model:* Smart reverse proxy with policy chains on request and response phases
* *Engine:* Reactive, non-blocking I/O via Vert.x/Netty with zero-copy buffer management
* *Components:*
** APIM Gateway -- runtime proxy enforcing policies
** Management API -- RESTful backend for configuration
** Management Console -- web UI for API producers
** Developer Portal -- web UI for API consumers
** Config Database (MongoDB/PostgreSQL)
** Analytics Database (Elasticsearch/OpenSearch)
* *Plugin system:* Extensible via policies, reporters, resources, repositories, entrypoints, endpoints
* *API Definition versions:* v2 (traditional HTTP proxy) and v4 (event-native, supports async protocols)

== Stateless Design

*Stateful by design.* The gateway requires:

* A config database (MongoDB/PostgreSQL)
* An analytics database (Elasticsearch/OpenSearch)
* Optionally Redis for distributed rate limiting and sync

With Redis Distributed Sync (4.10+), the gateway can continue serving APIs from Redis during database outages.
However, it is *not* a stateless proxy.

== Token Mediation / BFF Support

Evaluated against https://www.ietf.org/archive/id/draft-ietf-oauth-browser-based-apps-13.html[IETF draft-ietf-oauth-browser-based-apps]:

[cols="1,1,2"]
|===
| Variant | Supported? | Notes

| Token-Mediating Backend (encrypted cookie)
| No
| The APIM gateway validates tokens but does not initiate OAuth flows as a confidential client.

| BFF Proxy (server-side session)
| No
| The APIM gateway does not manage token sessions for browser-based SPAs.
|===

The APIM gateway does *not* act as an OIDC Relying Party -- it does not register with the IdP as a confidential client, does not redirect browsers to the IdP, and does not exchange authorization codes for tokens.
The APIM gateway only performs token *validation* (JWT signature verification or OAuth2 introspection on already-acquired tokens).

Gravitee Access Management (AM) is a separate product that acts as a full OAuth2/OIDC *provider* (like Keycloak), but this is an identity provider -- not the API gateway performing token mediation.

A community forum post from 2023 explicitly asks for BFF/Token Handler functionality and received zero answers.

=== Automatic Token Refresh

*Not supported* at the APIM gateway level. The gateway validates tokens via introspection but does not automatically refresh them.
Gravitee Access Management (AM) as an IdP supports the `refresh_token` grant, but the client must explicitly invoke it.

=== Step-Up Authentication (RFC 9470)

*Not supported* at the APIM gateway level. The gateway does not parse upstream `WWW-Authenticate` headers for `insufficient_user_authentication`, `acr_values`, `max_age`, or `scope` challenges.
It cannot trigger re-authorization with elevated authentication levels or additional scopes based on upstream responses.

Gravitee AM supports MFA step-up at the *login* level (triggering additional factors based on policy rules), but this is at the IdP level, not the API gateway level.
A https://github.com/gravitee-io/issues/issues/4031[GitHub issue #4031] requests FAPI ACR claim support, indicating this is not yet implemented.

== Forwarded Header / RFC 7239

*No RFC 7239 support found.* The gateway handles legacy `X-Forwarded-*` headers but there is no evidence of standardized `Forwarded` header generation or parsing per RFC 7239.

== Strengths

* Unique event-native architecture with Kafka/MQTT/Solace protocol mediation (EE)
* Full API lifecycle management (design, publish, secure, monitor, monetize)
* Reactive Vert.x/Netty engine for high-throughput non-blocking I/O
* Flexible deployment (self-hosted, hybrid, SaaS)
* Comprehensive policy engine (50+ built-in policies)
* Built-in developer portal
* Kubernetes Operator for CRD-based GitOps
* Gartner Magic Quadrant Leader (2025)

== Weaknesses

* *Open-core:* All async/event-native features, many reporters, Redis caching are Enterprise-only
* Low community signal (~389 GitHub stars)
* No HTTP/3 support
* No native WAF
* No BFF support
* No RFC 7239 support
* Heavy infrastructure requirements (MongoDB/PostgreSQL + Elasticsearch minimum)
* Management UI criticized as weak by some users
* Limited GraphQL support (no federation)
