# Long-lived worktree carries stale .plan script copies after a plan-marshall upgrade - refresh worktree scripts post-upgrade

This plan's worktree was created under plan-marshall 0.1.1166 and the plan later ran under
0.1.1172 (the bundle was upgraded mid-flight — see the 0.1.1172 landing commit). The
worktree's own `.plan/` script copies were NOT refreshed on upgrade, so they stayed pinned
at the 0.1.1166 revision. At phase-6 finalize, the current workflow invoked the
`review_completeness` script with the `--settled-bots` and `--triage-ran` flags that the
0.1.1172 script defines — but the stale 0.1.1166 copy in the worktree did not declare those
flags, so the invocation failed (argparse rejection). The only way past it was the documented
`force-done` escape hatch, which bypasses the review-completeness gate rather than satisfying
it.

Root cause: a long-lived worktree carries its own snapshot of the `.plan/` tooling, and a
mid-flight bundle upgrade advances the workflow docs (which call new flags) without advancing
the worktree's script copies. The workflow and the scripts it drives then disagree on the
argparse surface — new flag names against an old script body.

## Rule

- After a plan-marshall bundle upgrade, refresh the `.plan/` script copies in every
  still-open worktree before running any later-phase step (phase-6 review-completeness is the
  observed failure point). Do not assume a worktree created under the prior version carries
  the upgraded scripts.
- When a `manage-*` / workflow script invocation fails with an argparse rejection on a flag
  the current workflow doc clearly documents (e.g. `--settled-bots`, `--triage-ran`), suspect
  worktree/version skew FIRST — a stale script copy lacking a newly-added flag — before
  treating it as an invented-subcommand or a workflow bug. Re-sync the worktree scripts and
  retry rather than reaching straight for the `force-done` escape hatch.

## Prevention

Fold a worktree script-refresh into the post-upgrade reconciliation routine (alongside the
existing preflight + `manage-config sync-defaults` step): after landing a plan-marshall
upgrade, re-sync or recreate the `.plan/` script copies of any open worktrees so their
argparse surface matches the upgraded workflow docs. Reserve `force-done` for genuine
gate-inapplicability, not for papering over a version-skew invocation failure that a
worktree re-sync would fix cleanly.

## Impact

Cost a phase-6 review-completeness stall on this plan (PR #88), resolved only via the
`force-done` escape hatch — which advanced finalize without the review-completeness gate
actually running. Recurs on any plan whose worktree predates a plan-marshall upgrade that
adds or renames a script flag the later-phase workflow invokes.
