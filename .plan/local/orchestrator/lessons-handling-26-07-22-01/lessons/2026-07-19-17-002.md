# Deletion consumer-sweep must include the integration-tests module, not just main and unit tests

When the plan deleted the Plan-01 placeholder surface (the temporary `/api/*`
endpoints), the consumer sweep covered `src/main` and unit tests but missed the
`integration-tests` module. Stale ITs continued to assert against the now-deleted
endpoints and failed only later.

## Rule

- In a multi-module Maven project, deleting or replacing a production surface requires
  sweeping EVERY module for consumers — explicitly including `integration-tests`, which
  is easy to forget because it is a separate coordinator module, not part of the
  main/unit-test source tree.
- A consumer search scoped to the owning module's `src/main` + `src/test` is
  incomplete. Include integration-tests source, IT resources/fixtures, and any
  cross-module test that references the deleted API by path or symbol.
- Prefer a full-repo reference search for the deleted identifier/URL path over a
  module-local one before considering the deletion complete.

## Impact

Stale integration tests referencing deleted endpoints break the build after the
deletion looks complete locally. Recurs on any surface removal in a multi-module repo.
