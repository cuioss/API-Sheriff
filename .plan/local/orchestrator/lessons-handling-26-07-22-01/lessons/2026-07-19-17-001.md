# Warm marshalld build daemon masks cold test-classpath resolution gaps that only CI exposes

A locally-green build passed through the warm marshalld build daemon, but CI's cold
dependency resolution failed three rounds in a row. The daemon holds a warm,
already-resolved classpath, so test-scope dependencies that are present transitively
(or left over from a prior resolution) resolve fine locally even when they are not
declared where a clean/cold resolution needs them. Two distinct gaps hid this way:

- The `jjwt` 0.12 split artifacts (the `-impl` / `-jackson` runtime pieces) were not
  fully declared for the test classpath.
- The `smallrye-fault-tolerance-standalone` SPI provider was missing from the test
  classpath.

Both resolved from the daemon's warm cache and only surfaced on CI's cold build.

## Rule

- A green build from the warm build daemon is NOT evidence that dependencies are
  correctly declared. Warm-classpath success can mask missing test-scope declarations.
- Before trusting green on a dependency-affecting change, force a cold resolution
  (clean local build with no daemon warmth, or treat the first CI round as the real
  gate) so classpath gaps surface locally instead of after push.
- Split/multi-artifact dependencies (impl/api/jackson pieces, SPI providers) are the
  highest-risk class — declare every runtime/test piece explicitly rather than relying
  on transitive presence.

## Recurrence (2026-07-21, plan anchor-types-assets)

The same warm/stale-local-state-masks-CI failure recurred through a different
mechanism: a **stale `target/*-runner`** artifact. A `ConfigProducer` topology-alias
collection gap (upstream asset routes not collected into the route table) was masked
locally because `verify` reused an already-built `target/*-runner` from a prior run
**without a clean**, so the rebuilt-topology code path was never re-materialized
locally. CI's clean build re-materialized the runner and went red. The class is
identical to the warm-daemon case — a warm/stale local build artifact certifies code
that a cold/clean build proves broken.

**Broadened rule**: local staleness that can mask a CI-only defect is not limited to
the build daemon's classpath cache — it includes any reused build artifact
(`target/*-runner`, exploded image, incremental output). For any change that mutates
producer wiring / topology / reflection registration, run a **clean** build (or treat
the first CI round as the gate) before trusting green.

## Impact

Masked classpath gaps cost three CI convergence rounds on the original plan, and the
stale-runner variant was one of three consecutive CI re-settles on a later plan. The
failure mode recurs on any change validated only through warm daemon state or a reused
`target/*-runner` without a clean.
