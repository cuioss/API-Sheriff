# Auth-mandatoriness must be evaluated at the most-specific override level, not just the endpoint (ADR-0007)

An authorization defect slipped past the in-run gate and was caught by the PR
review bot against ADR-0007: an **endpoint-level** auth-mandatoriness check
masked **route-level** overrides. The code decided whether authentication was
mandatory by reading the endpoint's setting, but a route within that endpoint
could override mandatoriness to a weaker value — and because the check consulted
only the endpoint level, the route-level override was never applied. The result
was an auth requirement that read as enforced at the coarse level while a
finer-grained override silently relaxed (or should have tightened) it.

## Rule

When a policy attribute (here, auth-mandatoriness) is expressible at more than one
configuration level, evaluate it at the **most-specific level that applies to the
request**, honoring the override precedence ADR-0007 defines. Resolve the
effective value by walking from the most specific scope (route) outward
(endpoint / anchor / global) and taking the first explicit setting — do not read a
single coarse level and assume it is authoritative. A per-endpoint check that
ignores per-route overrides is a security bug, not a simplification.

## Prevention

For every override-capable security attribute, add a test that sets the coarse
level one way and a nested override the other way, and assert the *effective*
decision follows the override. The dangerous case is specifically "coarse says
mandatory, fine says optional" (and its inverse) — a test that only exercises a
single level will pass while the override path is broken. Make override resolution
a single shared helper so every consumer resolves precedence identically rather
than each re-reading one level.

## Impact

Applies to all multi-level policy resolution in api-sheriff governed by ADR-0007
(auth-mandatoriness today; CORS, rate limits, and any future per-route override
attribute by the same reasoning). Masking a finer-grained override behind a
coarser check defeats the layered-policy model the gateway's security posture
depends on.
