# Unit-green is not integrated: assert plan-mandated components are wired into the production request path

Two plan-mandated components — `SheriffMetrics` and `StreamAwareRetryGate` — were fully
implemented and unit-tested (all green) but were never wired into the production request
path. Unit-test success proved the units worked in isolation; it proved nothing about
whether the production code actually invokes them.

## Rule

- Unit-green is NOT integrated. A component can have 100% passing unit tests and still be
  dead code that no production path ever calls.
- For every plan-mandated component, add an assertion that it is actually reached on the
  production path: an integration test that exercises the real wiring, a CDI/injection
  assertion that the bean is produced and consumed, or an end-to-end test that observes
  the component's side effect (metric emitted, retry gate engaged).
- During finalize/verification, cross-check each deliverable's component against a
  producer→consumer trace, not just its unit-test file. "Tests pass" is not evidence of
  wiring.

## Impact

Two required behaviors (metrics, stream-aware retry) were silently absent from the running
gateway despite green unit tests. Recurs whenever a component is verified by unit tests
alone without a wiring/integration check.
