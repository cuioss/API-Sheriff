# SonarCloud coverage ingests only unit-test JaCoCo; cover IT-only edge logic at the unit level

Pipeline/edge code that was exercised only by integration tests showed roughly 13%
coverage on SonarCloud, because the `sonar-build` CI job runs no integration tests — it
ingests only the unit-test JaCoCo report. The coverage recovered once unit-level pipeline
tests were added.

## Root cause

SonarCloud coverage reflects only what the `sonar-build` job produces, and that job runs
unit tests only. Code covered exclusively by ITs is invisible to the coverage gate even
though it is genuinely tested.

## Rule

- Do not rely on integration tests to satisfy a coverage/quality gate that ingests only
  unit-test coverage. IT-covered code will report artificially low coverage and can fail
  the gate.
- For edge/pipeline logic that is naturally IT-shaped, add unit-level tests so the
  coverage the gate actually measures reflects reality — or configure the coverage
  aggregation to include the IT JaCoCo report if ITs are meant to count.
- When a coverage number looks implausibly low for code you know is tested, first check
  WHICH test tier the coverage tool ingests before assuming the code is untested.

## Impact

A false ~13% coverage reading on IT-covered pipeline code risked failing the Sonar gate.
Recurs for any code covered only by a test tier the coverage job does not run.
